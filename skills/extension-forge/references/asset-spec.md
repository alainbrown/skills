# Asset Spec — dimensions & composition map

All marketing assets are generated by the Remotion project in `demo/`, rendered in Docker, and
written to `docs/`. Each composition reuses the extension's **pure presentational components** with
mock props — one source of truth, no drift between product and demo.

## Composition → output map

| Remotion composition | Output file | Dimensions | Used for |
|---|---|---|---|
| `Demo` (video) | `docs/demo.mp4` | 1280×800 (or 1280×720) @ 30fps, ~15-25s | YouTube + CWS video, landing page |
| `Demo` → ffmpeg | `docs/demo.gif` | scaled ~640–960 wide, palette-optimized | README hero |
| `Screenshot1`–`Screenshot5` | `docs/store/screenshot-{1..5}.png` | **1280×800** | CWS listing screenshots (1–5) |
| `Marquee` | `docs/store/marquee.png` | **1400×560** | CWS marquee promo tile |
| `PromoTile` | `docs/store/promo-tile.png` | **440×280** | CWS small promo tile |
| `Thumbnail` | `docs/youtube-thumbnail.png` | **1280×720** | YouTube video thumbnail |

## Chrome Web Store official requirements (verify current rules at submission)

- **Store icon:** 128×128 PNG (also in the manifest `icons`).
- **Screenshots:** at least 1, up to 5. **1280×800** or 640×400 (1280×800 recommended). PNG or JPEG, no alpha for JPEG.
- **Small promo tile:** 440×280 PNG/JPEG (required for some placements).
- **Marquee promo tile:** 1400×560 PNG/JPEG (for the featured marquee).
- **Video:** a single YouTube URL (not a file upload) — that's what `demo.mp4` is for once uploaded.

## Manifest icon sizes

`16` (toolbar), `32` (Windows), `48` (extensions page), `128` (store + install). If `iconAutoGen`
is on, `scripts/render-icons.mjs` rasterizes `design/logo.svg` to all four into `public/icons/`.

## GIF optimization (ffmpeg, 2-pass palette)

```bash
ffmpeg -y -i docs/demo.mp4 \
  -vf "fps=20,scale=720:-1:flags=lanczos,split[a][b];[a]palettegen=max_colors=128[p];[b][p]paletteuse=dither=bayer:bayer_scale=5" \
  -loop 0 docs/demo.gif
```

Keep the GIF README-friendly (a few MB at most) — lower fps/width if it's heavy.

## Rendering rules

- `Config.setConcurrency(1)` for deterministic, scrub-safe renders.
- `Config.setVideoImageFormat('jpeg')` + `Config.setOverwriteOutput(true)`.
- Stills via `remotion still <root> <CompId> <out.png> --frame=<n>`.
- Pre-`ensureBrowser()` in the Docker image so the first `docker run` doesn't pay the Chromium download.
